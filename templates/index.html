// endpoints

Five clean endpoints.
Infinite emotional support.

Firebase-authenticated REST API — all endpoints require a Bearer token. Identity is derived server-side from the verified Firebase token.

POST
/api/companion/generate/
Send an emoji + thoughts and receive a warm, empathetic AI response from Luna. Pass optional conversation history for multi-turn context. Luna signals a resolved session with [SESSION_END] in her reply.
Body: emoji, thoughts Optional: history[] Auth: Bearer token Returns: MoodEntry object Content-Type: application/json
🤖 AI Powered
GET
/api/companion/history/
Retrieve mood journal entries for the authenticated user, ordered by most recent first. Each entry includes emoji, thoughts, Luna's full AI response, and timestamp.
Auth: Bearer token Returns: MoodEntry array Ordered: newest first
📔 Journal
GET
/api/accounts/me/
Get or update the authenticated user's profile. PATCH accepts full_name, phone_number, bio, date_of_birth, gender. Firebase identity fields are read-only.
Auth: Bearer token Returns: user profile PATCH supported
👤 Profile
DELETE
/api/accounts/delete-account/
Permanently deletes the authenticated user's Firebase identity and local account. Firebase deletion is attempted first; if it fails the local row is preserved and a 502 is returned.
Auth: Bearer token Irreversible
🗑️ Danger
GET
/api/companion/weekly-letter/
Generate a personalised weekly letter from Luna based on the last 7 days of the authenticated user's entries. Requires at least 2 entries in the past week. Returns a warm 3–4 paragraph letter, dominant mood emoji, and streak stats.
Auth: Bearer token Returns: letter + stats Min: 2 entries / week
💌 Weekly
// response schema

MoodEntry model

MoodEntry application/json
idintegerUnique identifier, auto-incremented
user_idstringInternal user PK — derived from Firebase token server-side, never accepted from the client
emojistringMood emoji selected by user (e.g. 😔 😊 😰)
thoughtsstringUser's written thoughts and feelings
ai_responsestringLuna's empathetic AI-generated response
created_atdatetimeISO 8601 timestamp of when entry was created

Production-ready stack.

Every layer chosen for speed, reliability, and developer experience.

💙
Flutter
Cross-platform mobile app powering Lueur's plant-growth UI, Firebase Auth sign-in, and real-time Luna conversations.
🐍
Django REST Framework
Battle-tested Python API framework with automatic serialization, browsable API, and clean URL routing.
⚡
GROQ — llama 3.1
Ultra-fast LLM inference using GROQ's LPU chip. 10–20× faster than traditional GPU inference.
🚂
Railway Deployment
Auto-deploy from GitHub with environment variables, zero-config HTTPS, and custom domain support.
📖
OpenAPI 3.0 Docs
Auto-generated Swagger UI + ReDoc documentation with live request testing and schema download.
🔒
Firebase Auth
Every endpoint is protected. Flutter clients authenticate via Firebase; the backend verifies ID tokens and derives identity server-side.
💾
Persistent Storage
Every mood entry is saved to the database. Full journal history with timestamps and unique IDs.

import requests

BASE    = "https://web-production-f8628
  .up.railway.app"

  f"{BASE}/api/companion/generate/",

    "history":  [],
  }
)
entry = res.json()
print(entry["ai_response"])
